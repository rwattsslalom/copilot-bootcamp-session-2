# Functional Requirements: To Do App

## FR-1: Add a Due Date to a Task

- Users shall be able to assign a due date to any task.
- The due date shall be selectable via a date picker or by entering a date manually.
- Users shall be able to update or remove the due date from an existing task.
- Tasks with a due date shall display the date visibly in the task list.

## FR-2: Edit a Task

- Users shall be able to edit the title and description of an existing task.
- Changes to a task shall be saved and immediately reflected in the task list.
- Users shall be able to cancel an edit without saving changes.

## FR-3: Sort Tasks

- Users shall be able to sort tasks by due date (ascending and descending).
- Users shall be able to sort tasks alphabetically by title (A–Z and Z–A).
- Users shall be able to sort tasks by creation date (newest and oldest first).
- The selected sort order shall persist for the duration of the user session.

## FR-4: Add Sub-Tasks

- Users shall be able to add one or more sub-tasks to any existing task.
- Sub-tasks shall be displayed nested under their parent task.
- Users shall be able to mark individual sub-tasks as complete independently of the parent task.
- Users shall be able to edit and delete sub-tasks.
- Completion of all sub-tasks shall optionally prompt the user to mark the parent task as complete.
